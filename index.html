Why we need git rebase?

Git rebase объединяет изменения двух веток. git rebase лучше чем git merge тем что он позволяет переписать историю ветки, как нам нужно.

Git fetch команда которая скачивает изменения из удаленного или локального репозитория в свой локальный репозиторий. git fetch хорош тем, что эта команда вносит актуальность локальному репозиторию без изменения его текущей ветки .

разница между git fetch и git pull в том что ,git fetch более гибкий он позволяет загружать информацию без изменения ветки, git pull пытается перестроить ветку под себя
